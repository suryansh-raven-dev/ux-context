# raven-ui Non-Negotiables — Audit Report

**Audit date:** 2026-04-17 &middot; **Against:** raven-ui Non-Negotiables Checklist v1.0 &middot; **Method:** static scan of `src/` + `index.ts` exports + `package.json` + theme sources.

**Legend** — `[x]` ships today (component/token + story) &middot; `[~]` partial (exists but missing variants, not exported, or provided via MUI only) &middot; `[ ]` missing from raven-ui.

### Headlines

- **Tokens** — core static families (color, typography, spacing, radius, shadow, motion) are in. Missing: z-index, breakpoints, density, opacity, granular focus-ring (width/offset), JSON export of tokens.
- **Components** — strong form/data-display/feedback coverage. Gaps in advanced overlays (Popover, ContextMenu, CommandPalette), streaming/AI infrastructure, and the entire Canvas primitive set.
- **Exported but not exposed** — several components live in `src/components/inputs/` but are not re-exported from `src/index.ts` (RavenButton, RavenButtonGroup, RavenSlider, RavenRating, RavenToggleButton, RavenTransferList, RavenFAB).
- **Theming** — `ravenAdminTheme` (light + dark) + `ravenNearMissTheme`. Missing: high-contrast mode, runtime tenant-token override API, JSON export.
- **Governance / docs** — stories + a11y addon + Chromatic + vitest. Missing: CONTRIBUTING, CHANGELOG, deprecation policy, migration guides, lint rules against hex literals, CI-gated axe.

## Priority roadmap (next five raven-ui issues)

Ranked by product pain — the gap most often forces local workarounds.

1. **Export the existing-but-hidden inputs.** RavenButton, RavenButtonGroup, RavenSlider, RavenRating, RavenToggleButton, RavenTransferList, RavenFAB — all live under `src/components/inputs/` but are absent from `src/index.ts`. Cheapest compliance win in the whole library.
2. **Ship `EmptyState`, `SkipLink`, `CategoryDot`.** Each is referenced by the guidelines as mandatory and each currently forces product code to inline the thing. All three are single-component PRs.
3. **Promote token families for `zIndex`, `breakpoint`, `density`, `opacity`, and full focus-ring.** These are what every composite component silently falls back to MUI defaults for today.
4. **Create a generic `FilterBar` + `RecordTable`.** `StatusFilterBar` is NMMS-flavored; the generic primitive it hides behind would unblock any future workflow product.
5. **Governance scaffolding.** Add `CHANGELOG.md`, `CONTRIBUTING.md`, and a single-line owner block in `README.md`. Hex-literal lint rule in the same PR — the grep pass we did for tokens should become an ESLint rule so regressions can't land.

## Methodology

- **Source of truth:** `src/index.ts` (what's actually exported) + `src/components/**/*.tsx` + `src/tokens/tokens.ts` + `src/theme/*.ts` + `package.json` + `.storybook/main.ts`.
- **Status rules:** `[x]` requires the component to be a named export in `src/index.ts` **and** have a `.stories.tsx` file. `[~]` = exists in `src/` but not exported, OR exported but missing states/variants, OR domain-flavored without a generic sibling. `[ ]` = no match in `src/`.
- **Out of scope:** runtime behavior verification, axe-core runs, bundle size measurements, visual regression diffs. These need a separate audit.
